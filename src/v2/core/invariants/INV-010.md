# INV-010: Saturn System Per-Body Interpolation Error Bound

## Statement

The maximum position error introduced by `core/` interpolation between fixture samples does not exceed the per-body cutover bar at any validated timestep, where each body's bar applies to that body's own fixture cadence.

## Per-Body Cutover Bars

Bars apply to cubic Hermite interpolation validated against Horizons truth using the cadence fixed for each body in the constants and boundary fixture contracts. These bars are calibrated from measured maxima in `tools/slice4-research/interpolation-report.md`, with explicit honest margins rather than a blanket "3x across the board."

| Body       | NAIF ID | Fixture cadence | Cutover bar | Measured max | Honest margin |
|------------|--------:|----------------:|------------:|-------------:|--------------:|
| Saturn     | 699     | `1 d`           | 1 km        | 0.202 km     | 4.9x          |
| Titan      | 606     | `12 h`          | 20 km       | 6.034 km     | 3.3x          |
| Rhea       | 605     | `3 h`           | 5 km        | 1.265 km     | 4.0x          |
| Iapetus    | 608     | `1 d`           | 2 km        | 0.619 km     | 3.2x          |
| Tethys     | 603     | `1 h`           | 1 km        | 0.284 km     | 3.5x          |
| Dione      | 604     | `3 h`           | 50 km       | 6.778 km     | 7.4x          |
| Mimas      | 601     | `1 h`           | 20 km       | 3.359 km     | 6.0x          |
| Enceladus  | 602     | `1 h`           | 5 km        | 0.857 km     | 5.8x          |

Source measurements: `tools/slice4-research/interpolation-report.md`.

## Runtime Check

```ts
assertInterpolationError(estimate: CanonicalState, truth: CanonicalState, bodyId: BodyId): void
```

- The runtime check reads from a unified interpolation-bar table that includes Slice 2, Slice 3, and Slice 4 bodies.
- The cadence per body is fixed in the constants module and is not passed at call time.
- In dev builds: throws `AssertError` with `invariantId: 'INV-010'` when position error between `estimate` and `truth` exceeds the cutover bar for `bodyId`.
- In production builds: emits a structured log entry with `invariantId: 'INV-010'` and continues.

Violation data shape:

```ts
{
  name: 'InvariantViolation',
  invariantId: 'INV-010',
  bodyId,
  measuredErrorKm,
  barKm,
  tdbSeconds,
}
```

## Failure Mode

- Dev: throws `AssertError` with `invariantId: 'INV-010'` at the first violating timestep.
- Prod: emits a structured log with `invariantId: 'INV-010'`, `bodyId`, `measuredErrorKm`, `barKm`, and `tdbSeconds`, then continues execution.

## Enforcement

- `core/interpolators` — extended for per-body cadence and called on interpolated Saturn-system states before return to any caller.
- Slice 4 cutover test suite — validates each body's interpolated state against Horizons truth at the validated timesteps for that body's cadence contract.

## Additive Invariant

INV-010 is additive. INV-008 (Slice 2 bars) and INV-009 (Slice 3 bars) remain in force unchanged. INV-010 adds the eight Saturn-system bodies without relaxing any prior invariant.

Slice 4's three independent `1 h` bodies validate the per-body cadence architecture established in Slice 3; the pattern extends cleanly without introducing a shared-grid requirement.

INV-005 (Propagation Drift Bounds) remains reserved for future propagator slices and is not exercised by Slice 4.
