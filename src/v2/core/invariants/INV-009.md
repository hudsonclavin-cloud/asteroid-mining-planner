# INV-009: Jupiter System Per-Body Interpolation Error Bound

## Statement

The maximum position error introduced by `core/` interpolation between fixture samples does not exceed the per-body cutover bar at any validated timestep, where each body's bar applies to that body's own fixture cadence.

## Per-Body Cutover Bars

Bars apply to cubic Hermite interpolation validated against Horizons truth using the cadence fixed for each body in the constants and boundary fixture contracts. These bars are calibrated from measured maxima in `tools/slice3-research/interpolation-report.md`, with explicit honest margins rather than a blanket "3x across the board."

| Body     | NAIF ID | Fixture cadence | Cutover bar | Measured max | Honest margin |
|----------|--------:|----------------:|------------:|-------------:|--------------:|
| Jupiter  | 599     | `1 d`           | 50 km       | 7.143 km     | 7.0x          |
| Io       | 501     | `1 h`           | 5 km        | 0.544 km     | 9.2x          |
| Europa   | 502     | `3 h`           | 20 km       | 4.504 km     | 4.4x          |
| Ganymede | 503     | `6 h`           | 20 km       | 6.610 km     | 3.0x          |
| Callisto | 504     | `12 h`          | 50 km       | 7.017 km     | 7.1x          |

Source measurements: `tools/slice3-research/interpolation-report.md`.

## Runtime Check

```ts
assertInterpolationError(estimate: CanonicalState, truth: CanonicalState, bodyId: BodyId): void
```

- The runtime check reads from a unified interpolation-bar table that includes both Slice 2 and Slice 3 bodies.
- The cadence per body is fixed in the constants module and is not passed at call time.
- In dev builds: throws `AssertError` with `invariantId: 'INV-009'` when position error between `estimate` and `truth` exceeds the cutover bar for `bodyId`.
- In production builds: emits a structured log entry with `invariantId: 'INV-009'` and continues.

Violation data shape:

```ts
{
  name: 'InvariantViolation',
  invariantId: 'INV-009',
  bodyId,
  measuredErrorKm,
  barKm,
  tdbSeconds,
}
```

## Failure Mode

- Dev: throws `AssertError` with `invariantId: 'INV-009'` at the first violating timestep.
- Prod: emits a structured log with `invariantId: 'INV-009'`, `bodyId`, `measuredErrorKm`, `barKm`, and `tdbSeconds`, then continues execution.

## Enforcement

- `core/interpolators` — extended for per-body cadence and called on interpolated Jupiter-system states before return to any caller.
- Slice 3 cutover test suite — validates each body's interpolated state against Horizons truth at the validated timesteps for that body's cadence contract.

## Additive Invariant

INV-009 is additive. INV-008 (Slice 2 bars) remains in force unchanged. INV-009 adds the five Jupiter-system bodies without relaxing any prior invariant.

INV-005 (Propagation Drift Bounds) remains reserved for future propagator slices and is not exercised by Slice 3.
