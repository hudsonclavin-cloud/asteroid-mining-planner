# INV-008: Interpolation Error Bound

## Statement

The maximum position error introduced by `core/` interpolation between fixture samples does not exceed the per-body cutover bar at any validated timestep.

## Per-Body Cutover Bars

Bars apply to cubic Hermite interpolation validated at 6-hour cadence against Horizons truth between daily fixture samples. Set at 3 × measured maximum Hermite error, rounded up to a clean value.

| Body    | NAIF ID | Cutover bar |
|---------|--------:|-------------|
| Sun     | 10      | 0.00002 km  |
| Mercury | 199     | 100 km      |
| Venus   | 299     | 1 km        |
| Earth   | 399     | 0.5 km      |
| Moon    | 301     | 20 km       |
| Mars    | 499     | 0.05 km     |

Source measurements: `tools/slice2-research/interpolation-report.md`.

## Runtime Check

```ts
assertInterpolationError(estimate: CanonicalState, truth: CanonicalState, bodyId: BodyId): void
```

- In dev builds: throws `AssertError` with `invariantId: 'INV-008'` when position error between `estimate` and `truth` exceeds the cutover bar for `bodyId`
- In production builds: emits a structured log entry with `invariantId: 'INV-008'` and continues

Violation data shape:

```ts
{
  name: 'InvariantViolation',
  invariantId: 'INV-008',
  bodyId,
  measuredErrorKm,
  barKm,
  tdbSeconds,
}
```

## Enforcement

- `core/interpolators` — called on every interpolated state before it is returned to any caller
- Slice 2 cutover test suite — validates every 6-hour intermediate timestep in the window `2026-05-01` to `2026-07-30` against Horizons truth

## Additive Invariant

INV-008 is additive. INV-001 through INV-007 continue to apply to all canonical state values without modification. INV-008 adds a constraint specific to the interpolation layer and does not relax any prior invariant.

INV-005 (Propagation Drift Bounds) is not exercised by Slice 2. INV-005 remains reserved for future propagator slices.
